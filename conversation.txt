AI-SDET PROJECT — FULL CONVERSATION TRANSCRIPT
================================================================================

Project: Autonomous AI-SDET pipeline that converts user stories into Cypress
test suites via LLM agents (LangChain/LangGraph + OpenAI proxy via
GreatLearning), runs them against a static HTML SPA (eclat-shop.html),
captures DOM snapshots, and (planned) self-heals failing tests. All traces
go to LangSmith.

This transcript covers two sessions. The first session built the foundation
(Steps 1-7); a compact summary of that work appears below. The second
session added the two-stage design (designer + generator), debugged
LangSmith tracing, and scaffolded the multi-story workflow.

USER: "Why is there a `cy.visit('/eclat-shop.html')` in page object ?"
--------------------------------------------------------------------------------
RESPONSE: Explained two POM schools:
  - School A — Page Object owns navigation (LoginPage.visit() pattern)
  - School B — Spec owns navigation, PO only models elements/actions

Recommended School B for this project because:
  1. App has exactly one route (single source of truth argument is weak)
  2. cy.visit triggers the afterEach → captureDom snapshot (test infra concern)
  3. Keeps POs free of Cypress lifecycle calls (easier Playwright port later)

    navigation primitive). Page Objects only model elements and actions
    that operate on an already-loaded page. Navigation is a test concern,
    not a page concern."
  - URL / NAVIGATION RULES: "Navigation belongs in the SPEC, not the Page
    Object. At the top of each describe(), use:
      beforeEach(() => { cy.visit('/eclat-shop.html'); });
    Do NOT add a visit() / open() / load() method to any Page Object."

inject "Existing Page Objects" block listing class names + methods so
the LLM extends instead of overwriting.

Cypress run mode: run all specs together at the end via
cypress/e2e/**/*.cy.js glob.

CONVENTIONS WORTH KEEPING
================================================================================

- ESM throughout ("type": "module"), except cypress.config.js (CommonJS).
- LLM output contract:
  * Designer:   { feature, story, cases: [...] }
  * Generator:  { files: [{ path, content }] }
- No cy.intercept (no backend).
- Always cy.visit('/eclat-shop.html') (no protocol/host hardcoded).
- No invented selectors (only DOM snapshot or catalogue entries).
- POM only — specs import from cypress/support/pages/.
- Page Objects MUST NOT call cy.visit. Navigation lives in the spec's
  beforeEach.
- Page Object reuse — generator receives an "Existing Page Objects"
  block listing class names + methods.
- DOM snapshot is source of truth — cypress/support/e2e.js runs
  cy.captureDom('homepage') after every test.
- Stories live in stories/<slug>.md.
- Slug-based file naming throughout (cypress/test-cases/<slug>.json,
  cypress/e2e/<slug>.cy.js).
