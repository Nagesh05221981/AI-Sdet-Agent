PAGE OBJECT PATTERN (MUST FOLLOW EXACTLY)
================================================================================
Every Page Object MUST use this exact structure:

  class PageName {
      elements = {
          buttonName: () => cy.get('#selector'),
          inputField: () => cy.get('#selector'),
          dynamicElement: (param) => cy.contains('.class', param),
      }

      // Action methods — perform user interactions
      clickSomething() {
          this.elements.buttonName().should('be.visible').click()
      }

      fillForm(email, password) {
          this.elements.emailInput().should('be.visible').clear().type(email)
          this.elements.passwordInput().should('be.visible').clear().type(password)
      }

      // Verification methods — contain assertions only
      verifySomethingVisible() {
          this.elements.something().should('be.visible')
      }
  }

  export default PageName

KEY RULES:
- elements = {} object with ARROW FUNCTIONS returning cy commands
- Parameterized elements accept parameters: (cat) => cy.contains('.fchip', cat)
- Action methods: named click*, open*, fill*, submit* — perform interactions
- Verify methods: named verify* — contain only assertions
- Action methods include .should('be.visible') before .click()/.type()
- Input actions use .clear().type(value) pattern
- export default ClassName (NOT named export)
- Page Objects MUST NOT call cy.visit()
- Do NOT use cy.submit() — click the .msubmit button inside the form
- Do NOT use cy.intercept()
