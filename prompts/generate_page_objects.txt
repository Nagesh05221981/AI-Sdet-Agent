NOVA / Eclat shop, a single-page static HTML app. You receive the full DOM
snapshot of the app. Your job is to create a complete set of Page Objects
covering every interactive area of the app, plus a BaseTest class that
spec files will import.

================================================================================
#1 RULE — READ THIS FIRST
================================================================================
Discover ALL selectors from the DOM snapshot. Do NOT invent selectors.
Only use IDs, classes, and text that actually exist in the DOM.

================================================================================
OUTPUT RULES (STRICT)
================================================================================
- Respond with a SINGLE JSON object and NOTHING ELSE.
- No prose, no markdown, no code fences, no comments outside the JSON.

  {
    "files": [
      { "path": "relative/path/to/file.js", "content": "valid javascript source" }
    ]
  }

================================================================================
FILES TO CREATE
================================================================================

1. cypress/support/pages/ShopPage.js
   Covers the main shopping area:
   - Getters: grid, product cards, product name/category/price, results info,
     filter chips, active filter, cart count, cart drawer, cart items, cart total,
     checkout button, search input
   - Action methods: filterBy(category), resetFilter(), search(term),
     addToCart(productName), openCart(), closeCart()

2. cypress/support/pages/AuthPage.js
   Covers authentication (login + signup):
   - Getters: auth modal, login tab, signup tab, login email/password fields,
     signup name/email/password fields, login message, signup message,
     user chip, username label, submit buttons
   - Action methods: openLoginModal(), openSignupModal(), switchToLogin(),
     switchToSignup(), fillLoginForm(email, password), fillSignupForm(name, email, password),
     submitLogin(), submitSignup(), seedUser(name, email, password) — seeds localStorage

3. cypress/support/BaseTest.js
   A helper class that spec files import for common setup:
   - visitShop() — calls cy.visit('/eclat-shop.html')
   - loadFixture(name) — loads and returns fixture data via cy.fixture()
   - seedUserAndVisit(name, email, password) — seeds nova_users in localStorage
     then visits the page (for login tests)
   - Exports instances of ShopPage and AuthPage so specs can do:
       import { shopPage, authPage, BaseTest } from '../support/BaseTest.js';

================================================================================
PAGE OBJECT RULES
================================================================================
- Page Objects have TWO kinds of members:
  1. GETTERS for element access (used in specs for assertions only):
       get grid() { return cy.get('#grid'); }
  2. ACTION METHODS for user interactions:
       filterBy(category) { this.filterChips.contains(category).click(); }
- Page Objects MUST NOT call cy.visit() — navigation lives in BaseTest.
- Use ES module syntax (import/export). No require().
- Discover selectors from the DOM snapshot provided below.
- Prefer IDs for unique elements, classes for repeating elements.
- Do NOT invent data-cy, data-test, or data-testid attributes.
- Do NOT use cy.submit() — form containers may be <div> not <form>.
  Click the submit button (class .msubmit) inside the form instead.

================================================================================
BASETEST RULES
================================================================================
- BaseTest is NOT a Cypress spec — it has no describe/it blocks.
- It exports helper functions and page object instances.
- Specs import from it:
    import { shopPage, authPage, visitShop, seedUserAndVisit } from '../support/BaseTest.js';
- seedUserAndVisit must set localStorage BEFORE cy.visit so the app reads
  the seeded user on page load. nova_users is a JSON OBJECT keyed by email:
    { "a@b.com": { "name": "Jane", "password": "pass123" } }

================================================================================
APP BEHAVIOUR
================================================================================
- There is NO backend. All data lives in JavaScript memory or localStorage.
- Do NOT use cy.intercept() — there are no network calls.
- Auth uses localStorage key 'nova_users' (object keyed by email).
- Cart state does NOT persist across page reloads.
- The grid renders synchronously on page load.

Return EXACTLY the JSON object. No prose. No code fences.
